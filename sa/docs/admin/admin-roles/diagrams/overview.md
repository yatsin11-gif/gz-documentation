## Роли в админке — визуальный TL;DR

Источник: [../requirements/feature-spec.md](../requirements/feature-spec.md). Схема — краткая суть, детали в спеке.

### User flow (что видит пользователь)

```mermaid
flowchart TD
    SA[Супер-админ] --> CREATE_A[Создать администратора]
    SA --> CREATE_C[Создать курьера<br/>та же форма:<br/>логин, пароль, статус]
    CREATE_A --> S2["Отметить разделы (множественный выбор):<br/>Станции / Пользователи / Партнёры"]
    S2 --> V{Хотя бы один раздел отмечен?}
    V -->|нет| ERR[Сохранение заблокировано]
    V -->|да| S3[Администратор создан с набором разделов]
    S3 --> EDIT[Данные и набор разделов<br/>после создания неизменны]
    EDIT --> L[Администратор логинится]
    L --> N["Боковое меню показывает<br/>только назначенные разделы<br/>(недоступные — полностью отсутствуют)"]
    N --> D{Прямой переход<br/>к неназначенному разделу?}
    D -->|да| BLOCK[Доступ отклонён]
    CREATE_C --> CC[Курьер создан<br/>статус по умолчанию: активен]
    CC --> CBLOCK[Блокировка курьера<br/>необратима]
    CC --> CL[Курьер логинится]
    CL --> CVIEW[Доступ к станциям и ПБ<br/>см. courier-management]
```

### Что внутри (pipeline)

```mermaid
flowchart LR
    REQ[Запрос к разделу] --> AUTH{Роль?}
    AUTH -->|супер-админ| ALLOW[Доступ разрешён ко всему]
    AUTH -->|админ| CHECK{Раздел в назначенном наборе?}
    CHECK -->|да| ALLOW
    CHECK -->|нет| DENY[Отклонено]
    AUTH -->|курьер| CCHECK{Раздел разрешён курьеру?}
    CCHECK -->|да| CALLOW[Доступ к станциям/ПБ]
    CCHECK -->|нет| DENY
```

### Список разделов

```mermaid
flowchart LR
    STATIONS["Станции:<br/>создание, архивирование"]
    USERS["Пользователи:<br/>создание, промоаккаунт,<br/>блокировка, инфо + история аренд"]
    PARTNERS["Партнёры:<br/>создание, редактирование,<br/>список, баланс/выплата/аренды"]
```

> Этап в конвейере: **Jira-ready** (2 issues) → **QA test cases** готовы. См. [../../../PROGRESS.md](../../../PROGRESS.md).

CRUD-логика разделов — в `admin/stations-management`, `admin/user-management`, `partners/partner-program`. Детальная функциональность курьера — в `admin/courier-management`.
