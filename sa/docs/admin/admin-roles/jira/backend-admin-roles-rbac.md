## Problem Statement

Сейчас нет модели ролей администраторов и серверной проверки доступа по разделам. Нужна модель «супер-админ» (полный доступ), «админ» (набор явно назначенных разделов) и «курьер» (фиксированный набор разрешённых разделов/действий), назначаемая при создании учётной записи.

## Scope

- модель ролей: супер-админ (полный доступ), админ (набор разделов из списка: «Станции», «Пользователи», «Партнёры»), курьер (фиксированный набор разрешённых разделов/действий, см. `sa/docs/admin/courier-management`)
- создание администратора только от имени супер-админа с обязательным множественным выбором хотя бы одного раздела
- создание курьера только от имени супер-админа (поля: логин, пароль, статус; статус по умолчанию «активен»); блокировка курьера необратима
- серверная проверка принадлежности раздела/эндпоинта набору, назначенному администратору или курьеру

## Out of Scope

- гранулярные права на уровне отдельных действий внутри раздела
- аудит-лог действий администраторов
- CRUD-логика внутри разделов «Станции», «Пользователи», «Партнёры» (отдельные фичи: `admin/stations-management`, `admin/user-management`, `partners/partner-program`)
- редактирование данных и набора разделов уже созданного администратора (подтверждено SA — функциональность не предусмотрена, эндпоинт редактирования не нужен)

## Implementation Notes

- affected modules: модуль аутентификации/авторизации администраторов `gozap-backend`
- constraints: нет
- known technical context: нет связанных foundation-задач, это базовая задача домена `admin`

## Acceptance Criteria

1. Только супер-админ может создать новую учётную запись администратора или курьера.
2. Создание администратора без хотя бы одного указанного раздела отклоняется.
3. Запросы администратора к разделам/эндпоинтам, не входящим в его назначенный набор, отклоняются на backend.
4. Супер-админ имеет доступ ко всем разделам без дополнительной проверки набора.
5. Backend не предоставляет эндпоинт для редактирования данных или набора разделов уже созданного администратора.
6. Backend поддерживает создание курьера с полями: логин, пароль, статус (по умолчанию «активен»).
7. Блокировка курьера необратима — backend не предоставляет возможность разблокировки.
