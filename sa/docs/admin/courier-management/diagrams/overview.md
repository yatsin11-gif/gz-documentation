## Управление курьерами — визуальный TL;DR

Источник: [../requirements/feature-spec.md](../requirements/feature-spec.md). Схема — краткая суть, детали в спеке.

### User flow (что видит пользователь)

```mermaid
flowchart TD
    SA[Супер-админ] --> CREATE[Создать курьера<br/>та же форма, что для админов<br/>логин, пароль, статус]
    SA --> CL[Список курьеров:<br/>имя, статус, контакты]
    CL --> EDIT[Редактировать данные/статус]
    CL --> BLOCK[Заблокировать курьера<br/>разблокировка невозможна]
    BLOCK --> ERR[Заблокированный курьер →<br/>ошибка при входе]
    CL --> ASSIGN[Назначить станции курьеру<br/>просмотр в карточке,<br/>удаление назначения]
    SA --> LOG[Журнал действий:<br/>курьер, станция, действие, время,<br/>статус, ПБ до/после, результат, коммент]

    C[Курьер: вход] --> SL[Список назначенных станций:<br/>код, адрес, кол-во ПБ,<br/>слоты, статус]
    SL --> SORT[Сортировка по кол-ву ПБ<br/>↑ ↓]
    SL --> CARD[Карточка станции<br/>адрес, код, зав. номер,<br/>фото, график работы<br/>только чтение]
    CARD --> ON{Станция онлайн?}
    ON -->|да| OPS[Операции с ПБ]
    ON -->|нет| NOOPS[Обслуживание недоступно]
    OPS --> DISP[Выдача ПБ без QR]
    OPS --> ALL[Выдача всех ПБ]
    OPS --> RET[Возврат ПБ]
    OPS --> ST[Статус операции:<br/>успех / ошибка / отмена / отказ]
    CARD --> CMT[Сообщить о проблеме<br/>комментарием]
```

### Что внутри (pipeline)

```mermaid
flowchart LR
    AUTH[Авторизация курьера] --> ROLE{Роль = курьер?}
    ROLE -->|да| ACTIVE{Статус = активен?}
    ROLE -->|нет| DENY1[Отклонено]
    ACTIVE -->|да| ACCESS[Доступ к назначенным<br/>станциям и ПБ]
    ACTIVE -->|нет/заблокирован| DENY2[Ошибка авторизации]
    ACCESS --> STDATA[Данные станции:<br/>адрес, код, зав. номер,<br/>фото, график работы]
    STDATA --> STATION{Станция онлайн<br/>и синхронизирована?}
    STATION -->|да| PBOPS[Операции с ПБ:<br/>выдача, выдача всех, возврат]
    STATION -->|нет| REJECT[Статус «ошибка/отказ»<br/>операция не теряется,<br/>не завершается успехом]
    PBOPS --> STATUS[Статус операции:<br/>успех/ошибка/отмена/отказ]
    STDATA --> CMT[Комментарий о проблеме]
    STATUS --> JOURNAL[Журнал действий<br/>для супер-админа]
    CMT --> JOURNAL
    SADMIN[Супер-админ] --> MANAGE[CRUD курьеров:<br/>создание в общей форме,<br/>редактирование, блокировка<br/>разблокировка невозможна]
    SADMIN --> ASSIGN[Назначение станций курьеру:<br/>назначить, просмотреть, удалить]
    SADMIN --> JOURNAL
```

> Этап в конвейере: **Jira-ready** (2 issues) → **QA test cases** готовы. См. [../../../PROGRESS.md](../../../PROGRESS.md).

> Вне итерации: разблокировка, тайм-лимит доступа.
