## Business Context

- Почему существует эта задача: нужен backend-контракт для создания/редактирования/блокировки курьеров супер-админом, для предоставления списка станций курьеру и для выполнения операций с ПБ (выдача, возврат).

## Repository Scope

- Target repository: `gozap-backend`
- Related repositories: `gozap-admin`

## Problem Statement

Нужен backend-контракт для роли курьера: управление учётными записями курьеров (создание, редактирование, блокировка) супер-админом, API списка станций для курьера (код, адрес, количество ПБ, свободные слоты, статус станции, сортировка), API карточки станции (адрес, код, заводской номер, фото, график работы) и API операций с ПБ (выдача без QR, выдача всех, возврат), с проверкой авторизации по роли.

## Scope

- CRUD учётной записи курьера (логин, пароль, статус; статус по умолчанию «активен»); создание в той же форме, что и для администраторов (см. `sa/docs/admin/admin-roles`)
- список курьеров для супер-админа (имя, статус, контакты)
- блокировка курьера: заблокированный курьер не проходит авторизацию (возвращается ошибка); разблокировка невозможна
- API списка станций для курьера: код станции, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию
- API карточки станции для курьера (только чтение): адрес станции, код станции, заводской номер станции, фото станции, график работы
- API операций с ПБ: выдача ПБ без QR-кода, выдача всех ПБ, возврат ПБ; отклонение операций на офлайн-станции
- авторизация: курьер видит только разрешённые данные, управление курьерами — только супер-админу

## Out of Scope

- привязка курьера к конкретным станциям
- разблокировка курьера
- логирование операций с ПБ
- редактирование/архивирование станции курьером

## Implementation Notes

- affected modules: модуль курьеров (новый), модуль станций (расширение для роли курьера), модуль ПБ-операций (новый), авторизация по ролям
- constraints: поля учётной записи курьера: логин, пароль, статус (по умолчанию «активен»); форма создания — та же, что для администраторов; разблокировка невозможна
- known technical context: зависит от сущности станции из `sa/docs/admin/stations-management` и от ролевой модели `sa/docs/admin/admin-roles`

## Acceptance Criteria

1. Супер-админ может создать учётную запись курьера (логин, пароль, статус по умолчанию «активен») через API; форма создания — та же, что для администраторов.
2. Супер-админ может редактировать данные и статус курьера через API.
3. Супер-админ может заблокировать курьера; заблокированный курьер не проходит авторизацию и получает ошибку; разблокировка невозможна.
4. Курьер получает список станций с полями: код, адрес, количество ПБ, свободные слоты, статус станции; с сортировкой по количеству ПБ.
5. Курьер получает карточку станции (только чтение) с полями: адрес станции, код станции, заводской номер станции, фото станции, график работы.
6. Курьер может выдать ПБ без QR-кода, выдать все ПБ и вернуть ПБ через API.
7. Операции с ПБ на офлайн-станции отклоняются.
8. Запросы на управление курьерами от не-супер-админа отклоняются; запросы курьера на административные функции отклоняются.
