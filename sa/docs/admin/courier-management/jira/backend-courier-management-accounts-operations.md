## Problem Statement

Нужен backend-контракт для роли курьера: управление учётными записями курьеров (создание, редактирование, блокировка) супер-админом, назначение курьеру станций (назначить/просмотреть/удалить), API списка назначенных станций для курьера (код, адрес, количество ПБ, свободные слоты, статус станции, сортировка), API карточки станции (адрес, код, заводской номер, фото, график работы), API операций с ПБ (выдача без QR, выдача всех, возврат) с верхнеуровневыми статусами, API комментария курьера о проблеме и журнал действий для супер-админа, с проверкой авторизации по роли.

## Scope

- CRUD учётной записи курьера (логин, пароль, статус; статус по умолчанию «активен»); создание в той же форме, что и для администраторов (см. `sa/docs/admin/admin-roles`)
- список курьеров для супер-админа (имя, статус, контакты)
- блокировка курьера: заблокированный курьер не проходит авторизацию (возвращается ошибка); разблокировка невозможна
- назначение станций курьеру супер-админом: назначить список станций, вернуть назначенные станции в карточке курьера, удалить назначение
- API списка станций для курьера: только назначенные станции; код станции, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию
- API карточки станции для курьера (только чтение): адрес станции, код станции, заводской номер станции, фото станции, график работы
- API операций с ПБ: выдача ПБ без QR-кода, выдача всех ПБ, возврат ПБ; верхнеуровневые статусы операции («успех», «ошибка», «отмена», «отказ»); отклонение операций на офлайн/несинхронизированной станции (понятный статус, операция не теряется и не завершается успехом)
- API комментария курьера о проблеме по станции (повреждена, не работает, не совпадает количество ПБ)
- журнал действий для супер-админа: айди курьера, айди станции, действие, время, статус станции, количество ПБ до/после операции, результат (успех/ошибка) и причина, комментарий курьера при ошибке
- авторизация: курьер видит только назначенные ему станции и разрешённые данные, управление курьерами и журнал — только супер-админу

## Out of Scope

- разблокировка курьера
- редактирование/архивирование станции курьером

## Implementation Notes

- affected modules: модуль курьеров (новый), назначение станций курьеру (новый), модуль станций (расширение для роли курьера), модуль ПБ-операций со статусами (новый), журнал действий (новый), авторизация по ролям
- constraints: поля учётной записи курьера: логин, пароль, статус (по умолчанию «активен»); форма создания — та же, что для администраторов; разблокировка невозможна
- known technical context: зависит от сущности станции из `sa/docs/admin/stations-management` и от ролевой модели `sa/docs/admin/admin-roles`

## Acceptance Criteria

1. Супер-админ может создать учётную запись курьера (логин, пароль, статус по умолчанию «активен») через API; форма создания — та же, что для администраторов.
2. Супер-админ может редактировать данные и статус курьера через API.
3. Супер-админ может заблокировать курьера; заблокированный курьер не проходит авторизацию и получает ошибку; разблокировка невозможна.
4. Супер-админ может назначить курьеру список станций, получить назначенные станции в карточке курьера и удалить назначение.
5. Курьер получает список только назначенных ему станций с полями: код, адрес, количество ПБ, свободные слоты, статус станции; с сортировкой по количеству ПБ; неназначенные станции курьеру не возвращаются.
6. Курьер получает карточку станции (только чтение) с полями: адрес станции, код станции, заводской номер станции, фото станции, график работы.
7. Курьер может выдать ПБ без QR-кода, выдать все ПБ и вернуть ПБ через API; каждая операция получает верхнеуровневый статус («успех», «ошибка», «отмена», «отказ»).
8. Операции с ПБ на офлайн/несинхронизированной станции не завершаются успехом: операция получает понятный статус и не теряется.
9. Курьер может отправить комментарий о проблеме по станции; комментарий сохраняется и доступен супер-админу в журнале.
10. Каждая операция курьера с ПБ фиксируется в журнале действий с полями: айди курьера, айди станции, действие, время, статус станции, ПБ до/после, результат и причина, комментарий при ошибке; журнал доступен только супер-админу.
11. Запросы на управление курьерами (включая назначение станций и журнал) от не-супер-админа отклоняются; запросы курьера на административные функции и к неназначенным станциям отклоняются.
