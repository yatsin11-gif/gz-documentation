## Problem Statement

Нужен backend-контракт для роли курьера: управление учётными записями курьеров (создание, редактирование, блокировка) супер-админом, назначение курьеру станций (назначить/просмотреть/удалить), API списка назначенных станций для курьера (код, адрес, количество ПБ, свободные слоты, статус станции, сортировка), API карточки станции (адрес, код, заводской номер, фото, график работы), API забора ПБ (ввод количества, мин 1 — макс все доступные) и добавления ПБ (двухшаговый «начать»/«завершить» с фиксацией времени) с верхнеуровневыми статусами, API комментария курьера к действию и журнал действий для супер-админа, с проверкой авторизации по роли.

## Scope

- CRUD учётной записи курьера (логин, пароль, статус; статус по умолчанию «активен»); создание в той же форме, что и для администраторов (см. `sa/docs/admin/admin-roles`)
- список курьеров для супер-админа (имя, статус, контакты)
- блокировка курьера: заблокированный курьер не проходит авторизацию (возвращается ошибка); разблокировка невозможна
- назначение станций курьеру супер-админом: назначить список станций, вернуть назначенные станции в карточке курьера, удалить назначение
- API списка станций для курьера: только назначенные станции; код станции, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию
- API карточки станции для курьера (только чтение): адрес станции, код станции, заводской номер станции, фото станции, график работы
- API забора ПБ: курьер вводит количество ПБ (минимум 1, максимум — все доступные на станции на момент запроса); выдача без QR-кода; статус «успех» при выдаче запрошенного количества, «ошибка» при уходе станции в офлайн в процессе; операция только на прикреплённой к курьеру станции
- API добавления ПБ: «начать добавление» (фиксация времени начала) и «завершить добавление» (фиксация времени завершения); завершение возможно только если добавление начато этим курьером на этой станции; операция только на прикреплённой станции
- верхнеуровневые статусы операции («успех», «ошибка», «отмена», «отказ»); отклонение операций на офлайн/несинхронизированной станции (понятный статус, операция не теряется и не завершается успехом)
- API комментария курьера к действию при ошибке и о проблеме по станции (повреждена, не работает, не совпадает количество ПБ)
- журнал действий для супер-админа: логин курьера, id станции, действие (забор/добавление ПБ), дата/время действия (для добавления — время завершения), статус станции, количество ПБ до/после операции, статус действия, комментарий курьера (при наличии)
- авторизация: курьер видит только назначенные ему станции и разрешённые данные, управление курьерами и журнал — только супер-админу

## Out of Scope

- разблокировка курьера
- редактирование/архивирование станции курьером

## Implementation Notes

- affected modules: модуль курьеров (новый), назначение станций курьеру (новый), модуль станций (расширение для роли курьера), модуль ПБ-операций со статусами — забор по количеству и добавление «начать»/«завершить» (новый), журнал действий (новый), авторизация по ролям
- constraints: поля учётной записи курьера: логин, пароль, статус (по умолчанию «активен»); форма создания — та же, что для администраторов; разблокировка невозможна
- known technical context: зависит от сущности станции из `sa/docs/admin/stations-management` и от ролевой модели `sa/docs/admin/admin-roles`

## Acceptance Criteria

1. Супер-админ может создать учётную запись курьера (логин, пароль, статус по умолчанию «активен») через API; форма создания — та же, что для администраторов.
2. Супер-админ может редактировать данные и статус курьера через API.
3. Супер-админ может заблокировать курьера; заблокированный курьер не проходит авторизацию и получает ошибку; разблокировка невозможна.
4. Супер-админ может назначить курьеру список станций, получить назначенные станции в карточке курьера и удалить назначение.
5. Курьер получает список только назначенных ему станций с полями: код, адрес, количество ПБ, свободные слоты, статус станции; с сортировкой по количеству ПБ; неназначенные станции курьеру не возвращаются.
6. Курьер получает карточку станции (только чтение) с полями: адрес станции, код станции, заводской номер станции, фото станции, график работы.
7. Забор ПБ: курьер вводит количество (минимум 1, максимум — все доступные на момент запроса), выдача без QR-кода; при выдаче запрошенного количества статус «успех», при уходе станции в офлайн в процессе — «ошибка»; операция возможна только на прикреплённой станции.
8. Добавление ПБ: «начать добавление» фиксирует время начала, «завершить добавление» фиксирует время завершения; завершение возможно только если добавление начато этим курьером на этой станции.
9. Каждая операция получает верхнеуровневый статус («успех», «ошибка», «отмена», «отказ»); операции на офлайн/несинхронизированной станции не завершаются успехом: операция получает понятный статус и не теряется.
10. Курьер может добавить комментарий к действию при ошибке и сообщить о проблеме по станции; комментарий сохраняется и доступен супер-админу в журнале.
11. Каждая операция курьера с ПБ фиксируется в журнале действий с полями: логин курьера, id станции, действие (забор/добавление), дата/время действия (для добавления — время завершения), статус станции, ПБ до/после, статус действия, комментарий (при наличии); журнал доступен только супер-админу.
12. Запросы на управление курьерами (включая назначение станций и журнал) от не-супер-админа отклоняются; запросы курьера на административные функции и к неназначенным станциям отклоняются.
