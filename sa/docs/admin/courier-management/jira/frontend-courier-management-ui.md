## Problem Statement

В `gozap-admin` отсутствует интерфейс для роли курьера (список станций, карточка, операции с ПБ) и раздел управления курьерами для супер-админа (список, редактирование, блокировка).

## Scope

- вход курьера в систему (видит только разрешённые разделы и действия)
- список станций для курьера (код, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию)
- карточка станции для курьера (только чтение: адрес станции, код станции, заводской номер станции, фото станции, график работы; разрешённые действия)
- операции с ПБ: выдача без QR-кода, выдача всех ПБ, возврат ПБ; недоступность операций на офлайн-станции
- список курьеров для супер-админа (имя, статус, контакты, действия)
- форма создания курьера (та же форма, что для создания администраторов; поля: логин, пароль, статус по умолчанию «активен»)
- форма редактирования данных и статуса курьера для супер-админа
- блокировка курьера из интерфейса супер-админа; разблокировка невозможна

## Out of Scope

- редактирование/архивирование станции курьером
- привязка курьера к конкретным станциям
- разблокировка курьера

## Implementation Notes

- affected modules: раздел курьера в `gozap-admin` (новый), раздел управления курьерами для супер-админа (новый)
- constraints: курьер не должен видеть административные разделы; раздел управления курьерами доступен только супер-админу
- known technical context: зависит от API из `backend-courier-management-accounts-operations.md`

## Acceptance Criteria

1. Курьер входит в систему и видит только разрешённые разделы (без административных функций).
2. Курьер видит список всех станций с кодом, адресом, количеством ПБ, свободных слотов и статусом станции; может сортировать по количеству ПБ.
3. Курьер может открыть карточку станции и увидеть: адрес, код, заводской номер, фото, график работы (без возможности редактировать/архивировать).
4. Курьер может выполнить операции с ПБ: выдача без QR, выдача всех, возврат.
5. На офлайн-станции операции с ПБ недоступны.
6. Супер-админ видит список курьеров с именем, статусом, контактами и действиями.
7. Супер-админ может редактировать данные и статус курьера.
8. Супер-админ может заблокировать курьера.
