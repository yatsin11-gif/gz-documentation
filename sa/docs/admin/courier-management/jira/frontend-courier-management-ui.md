# Jira Task Spec

- Project: `GOZAP`
- Issue type: Task
- Summary: FRONT - Courier Management. Интерфейс курьера и управление курьерами для супер-админа

- Labels: `ai-ready`, `frontend`, `admin`, `courier`

## Business Context

- Почему существует эта задача: курьеру нужен интерфейс для просмотра назначенных станций, операций с ПБ и отправки комментария о проблеме, а супер-админу — интерфейс для управления курьерами (список, редактирование, блокировка, назначение станций, журнал действий).

## Repository Scope

- Target repository: `gozap-admin`
- Related repositories: `gozap-backend`

## Problem Statement

В `gozap-admin` отсутствует интерфейс для роли курьера (список назначенных станций, карточка, операции с ПБ, комментарий о проблеме) и раздел управления курьерами для супер-админа (список, редактирование, блокировка, назначение станций, журнал действий).

## Scope

- вход курьера в систему (видит только разрешённые разделы и действия)
- список станций для курьера (только назначенные; код, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию)
- карточка станции для курьера (только чтение: адрес станции, код станции, заводской номер станции, фото станции, график работы; разрешённые действия)
- операции с ПБ: выдача без QR-кода, выдача всех ПБ, возврат ПБ; отображение статуса операции («успех», «ошибка», «отмена», «отказ»); недоступность операций на офлайн-станции
- отправка комментария о проблеме по станции (повреждена, не работает, не совпадает количество ПБ)
- список курьеров для супер-админа (имя, статус, контакты, действия)
- форма создания курьера (та же форма, что для создания администраторов; поля: логин, пароль, статус по умолчанию «активен»)
- форма редактирования данных и статуса курьера для супер-админа
- блокировка курьера из интерфейса супер-админа; разблокировка невозможна
- назначение станций курьеру в карточке курьера: назначить, просмотреть назначенные, удалить назначение
- журнал действий для супер-админа (айди курьера, станции, действие, время, статус станции, ПБ до/после, результат и причина, комментарий курьера)

## Out of Scope

- редактирование/архивирование станции курьером
- разблокировка курьера

## Implementation Notes

- affected modules: раздел курьера в `gozap-admin` (новый), раздел управления курьерами для супер-админа (новый)
- constraints: курьер не должен видеть административные разделы; раздел управления курьерами доступен только супер-админу
- known technical context: зависит от API из `backend-courier-management-accounts-operations.md`

## Acceptance Criteria

1. Курьер входит в систему и видит только разрешённые разделы (без административных функций).
2. Курьер видит список только назначенных ему станций с кодом, адресом, количеством ПБ, свободных слотов и статусом станции; может сортировать по количеству ПБ.
3. Курьер может открыть карточку станции и увидеть: адрес, код, заводской номер, фото, график работы (без возможности редактировать/архивировать).
4. Курьер может выполнить операции с ПБ: выдача без QR, выдача всех, возврат; статус операции отображается пользователю.
5. На офлайн-станции операции с ПБ недоступны/не завершаются успехом.
6. Курьер может отправить комментарий о проблеме по станции.
7. Супер-админ видит список курьеров с именем, статусом, контактами и действиями.
8. Супер-админ может редактировать данные и статус курьера.
9. Супер-админ может заблокировать курьера.
10. Супер-админ может назначить курьеру станции, увидеть назначенные станции в карточке курьера и удалить назначение.
11. Супер-админ может открыть журнал действий и увидеть записи операций с полями: айди курьера, станции, действие, время, статус станции, ПБ до/после, результат и причина, комментарий курьера.
