## Проблема

- В системе нет роли курьера — сотрудника, который обслуживает станции: забирает и добавляет powerbank (ПБ), контролирует наличие ПБ и свободных слотов.
- Супер-админ не может создавать учётные записи курьеров, редактировать их данные, менять статус и блокировать доступ.
- Нет интерфейса, в котором курьер видит только разрешённые ему данные и действия.

## Цель

- Дать курьеру интерфейс для входа в систему, просмотра списка назначенных ему станций (код, адрес, количество ПБ, свободные слоты, статус станции), просмотра карточки станции (адрес, код, заводской номер, фото, график работы — только чтение) и выполнения операций с ПБ: забор ПБ из станции (курьер вводит количество, мин 1 — макс все доступные) и добавление ПБ в станцию (двухшаговый процесс «начать»/«завершить» добавление).
- Дать курьеру возможность сообщить о проблеме по станции (станция повреждена, не работает, не совпадает количество ПБ) через комментарий.
- Дать супер-админу возможность управлять курьерами: видеть список курьеров, редактировать данные и статус, блокировать курьера, назначать курьеру станции и удалять назначение.
- Дать супер-админу журнал действий курьеров по операциям с ПБ.

## Не входит в текущую итерацию

- Редактирование или архивирование станции курьером (курьер видит станцию только на чтение).
- Временное ограничение доступа курьера (тайм-лимит) — доступ не ограничен по времени.
- Создание курьера обычным администратором (только супер-админ).
- Разблокировка заблокированного курьера.

## Триггер и аудитория

- Точка входа / триггер: супер-админ создаёт учётную запись курьера; курьер входит в `gozap-admin` под своими учётными данными и видит только разрешённые разделы.
- Основная аудитория: курьер (операции со станциями и ПБ), супер-админ (управление курьерами).

## Целевое поведение продукта

### Сценарий

1. Супер-админ создаёт учётную запись курьера (логин, пароль, статус; статус по умолчанию «активен») в той же форме, что используется для создания администраторов (см. `sa/docs/admin/admin-roles`), при необходимости редактирует данные или блокирует курьера, а также вручную назначает курьеру список станций и может удалить назначение (назначенные станции видны в карточке курьера).
2. Курьер входит в систему под своей учётной записью и видит только доступные ему разделы и действия; срок действия доступа не ограничен тайм-лимитом.
3. Курьер видит список назначенных ему станций: код станции, адрес, количество ПБ, свободные слоты, статус станции; может сортировать список по количеству ПБ по убыванию/возрастанию.
4. Курьер открывает карточку станции и видит: адрес станции, код станции, заводской номер станции, фото станции, график работы (только чтение, без возможности редактировать или архивировать); в карточке доступны только разрешённые действия.
5. Забор ПБ: курьер на странице станции нажимает кнопку выдачи и вводит количество ПБ, которое хочет забрать (минимум 1, максимум — все доступные на станции на момент запроса). Если запрошенное количество успешно выдано — статус действия «успех»; если станция ушла в офлайн в процессе — статус «ошибка». Забор ПБ возможен только со станции, к которой курьер прикреплён, и только если станция не в офлайне.
6. Добавление ПБ: курьер нажимает кнопку «начать добавление пауэрбанков» (фиксируется время начала), затем «завершить добавление пауэрбанков» (фиксируется время завершения). Кнопка «завершить» доступна только если добавление было начато этим курьером на этой станции. Добавление возможно только на станцию, к которой курьер прикреплён, и только если станция не в офлайне.
7. Если при заборе или добавлении ПБ получена ошибка, курьер может добавить комментарий к действию; при проблеме со станцией (повреждена, не работает, не совпадает количество ПБ) сообщает о ней через комментарий.
8. Операция получает верхнеуровневый статус: «успех», «ошибка», «отмена» или «отказ». При ошибке операция не завершается успешно. Станцию в офлайне обслуживать нельзя: если станция недоступна или данные не синхронизированы, операция получает понятный статус и не теряется, но не может быть завершена как успешная.
9. Супер-админу доступен журнал действий: логин курьера, id станции, совершённое действие (забор/добавление ПБ), дата/время действия (для добавления — время завершения), статус станции, количество ПБ до и после операции, статус действия, комментарий курьера (при наличии).

### Правила результата

- Курьер должен видеть только разрешённые ему разделы и действия, без доступа к административным функциям.
- Курьер должен видеть только назначенные ему станции.
- Карточка станции для курьера — только чтение, без редактирования и архивирования.
- Заблокированный курьер при попытке авторизации должен получать ошибку.
- Станцию в офлайне обслуживать нельзя.
- Каждая операция с ПБ должна получать верхнеуровневый статус («успех», «ошибка», «отмена», «отказ») и попадать в журнал действий.
- Супер-админ должен видеть список всех курьеров с именем, статусом, контактами и действиями, а также назначать/удалять станции курьеру и просматривать журнал действий.

## Бизнес-правила

- Курьер — отдельная роль в системе, которая видит только разрешённые разделы и не может выполнять действия вне своей роли.
- Учётная запись курьера содержит поля: логин, пароль, статус. Статус по умолчанию — «активен».
- Супер-админ создаёт курьера в той же форме, что и остальных администраторов (см. `sa/docs/admin/admin-roles`).
- Разблокировка курьера невозможна: заблокированный курьер не может быть разблокирован.
- Курьер видит только станции, назначенные ему супер-админом.
- Супер-админ вручную назначает курьеру список станций и может удалить назначение; назначенные станции отображаются в карточке курьера.
- В списке станций отображаются: код станции, адрес, количество ПБ, свободные слоты, статус станции.
- Список станций для курьера поддерживает сортировку по количеству ПБ в обе стороны (по убыванию/возрастанию).
- Карточка станции для курьера содержит: адрес станции, код станции, заводской номер станции, фото станции, график работы.
- Курьер видит в карточке станции только разрешённые действия.
- Курьер может выполнять операции с ПБ только со станцией, к которой он прикреплён (назначена), и только если станция не в офлайне.
- Забор ПБ: курьер по нажатию кнопки выдачи вводит количество ПБ (минимум 1, максимум — все доступные на станции на момент запроса); выдача не использует QR-код пользовательского сценария. Если запрошенное количество выдано — статус «успех»; если станция ушла в офлайн в процессе — статус «ошибка».
- Добавление ПБ: двухшаговый процесс — «начать добавление пауэрбанков» (фиксируется время начала) и «завершить добавление пауэрбанков» (фиксируется время завершения). Кнопка «завершить» доступна только если добавление было начато этим курьером на этой станции.
- При ошибке операции (забор или добавление) курьер может добавить комментарий к действию.
- Курьер может сообщить о проблеме по станции (повреждена, не работает, не совпадает количество ПБ) через комментарий.
- Каждая операция имеет верхнеуровневый статус: «успех», «ошибка», «отмена», «отказ». При ошибке операция не завершается успешно; операция на офлайн-станции или при несинхронизированных данных получает понятный статус, не теряется, но не может завершиться успехом.
- Все операции курьера с ПБ фиксируются в журнале действий, доступном супер-админу: логин курьера, id станции, действие (забор/добавление ПБ), дата/время действия (для добавления — время завершения), статус станции, количество ПБ до и после операции, статус действия, комментарий курьера (при наличии).
- Супер-админ может редактировать данные курьера и менять его статус.
- Блокировка курьера супер-админом приводит к тому, что при авторизации заблокированный курьер получает ошибку.

## Пограничные сценарии

- Заблокированный курьер пытается войти в систему — должен получить ошибку авторизации.
- Курьер пытается обслужить станцию в офлайне — операция недоступна.
- Курьер при заборе вводит количество больше доступного на станции — количество ограничено максимумом (все доступные ПБ).
- Курьер при заборе вводит количество меньше 1 (0 или отрицательное) — операция не выполняется (минимум 1).
- Курьер пытается забрать ПБ со станции, к которой не прикреплён.
- Курьер пытается нажать «завершить добавление пауэрбанков» без предварительного «начать добавление» на этой станции — кнопка недоступна.
- Станция уходит в офлайн в процессе забора ПБ — статус действия «ошибка».
- Курьер обнаруживает несовпадение количества ПБ на станции и сообщает об этом через комментарий.
- Супер-админ удаляет назначение станции курьеру, у которого станция открыта в активной сессии.
- Супер-админ блокирует курьера, который в данный момент авторизован в системе.
- Супер-админ редактирует данные курьера параллельно с активной сессией курьера.
