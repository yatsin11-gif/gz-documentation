## Список задач

### Task 1

- Title: BE. Courier Management. Учётные записи курьеров, список станций и операции с ПБ
- Repository: `gozap-backend`
- Owner role: Backend engineer
- Зачем нужна задача: нужен backend-контракт для управления курьерами супер-админом, предоставления списка станций курьеру и выполнения операций с ПБ.
- Scope:
  - CRUD учётной записи курьера (логин, пароль, статус; статус по умолчанию «активен»); создание в той же форме, что и для администраторов (см. `sa/docs/admin/admin-roles`)
  - список курьеров для супер-админа (имя, статус, контакты)
  - блокировка курьера: заблокированный курьер не проходит авторизацию; разблокировка невозможна
  - назначение станций курьеру супер-админом: назначить список станций, вернуть назначенные станции в карточке курьера, удалить назначение
  - API списка станций для курьера: только назначенные станции; код станции, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ по убыванию/возрастанию
  - API карточки станции для курьера (только чтение): адрес станции, код станции, заводской номер станции, фото станции, график работы
  - API операций с ПБ: выдача ПБ без QR-кода, выдача всех ПБ, возврат ПБ; верхнеуровневые статусы операции («успех», «ошибка», «отмена», «отказ»); отклонение операций на офлайн/несинхронизированной станции
  - API комментария курьера о проблеме по станции (повреждена, не работает, не совпадает количество ПБ)
  - журнал действий для супер-админа: айди курьера, айди станции, действие, время, статус станции, ПБ до/после операции, результат и причина, комментарий курьера при ошибке
  - авторизация: курьер видит только назначенные ему станции и разрешённые данные, управление курьерами и журнал — только супер-админу
- Out of scope:
  - разблокировка курьера
- Dependencies:
  - `sa/docs/admin/admin-roles` (ролевая модель)
  - `sa/docs/admin/stations-management` (сущность станции)
- Risks:
  - нет
- Definition of done:
  - супер-админ может создать, редактировать и заблокировать курьера, назначить/удалить станции курьеру через API
  - курьер получает список только назначенных станций с количеством ПБ, свободных слотов и статусом станции, с сортировкой по количеству ПБ
  - карточка станции возвращает: адрес, код, заводской номер, фото, график работы
  - курьер может выполнить операции с ПБ (выдача, выдача всех, возврат) через API; каждая операция получает статус; офлайн-станция отклоняет операции
  - курьер может отправить комментарий о проблеме; операции фиксируются в журнале действий, доступном супер-админу

### Task 2

- Title: FE. Courier Management. Интерфейс курьера и управление курьерами для супер-админа
- Repository: `gozap-admin`
- Owner role: Frontend engineer
- Зачем нужна задача: курьеру нужен интерфейс для работы со станциями и ПБ, а супер-админу — интерфейс для управления курьерами.
- Scope:
  - вход курьера в систему (видит только разрешённые разделы)
  - список станций для курьера (только назначенные; код, адрес, количество ПБ, свободные слоты, статус станции; сортировка по количеству ПБ)
  - карточка станции для курьера (только чтение: адрес, код, заводской номер, фото, график работы; разрешённые действия)
  - операции с ПБ: выдача без QR-кода, выдача всех, возврат; отображение статуса операции; недоступность операций на офлайн-станции
  - отправка комментария о проблеме по станции (повреждена, не работает, не совпадает количество ПБ)
  - список курьеров для супер-админа (имя, статус, контакты, действия)
  - форма создания курьера (та же форма, что для создания администраторов; поля: логин, пароль, статус)
  - форма редактирования данных и статуса курьера для супер-админа
  - блокировка курьера из интерфейса супер-админа
  - назначение станций курьеру в карточке курьера: назначить, просмотреть назначенные, удалить назначение
  - журнал действий для супер-админа (айди курьера, станции, действие, время, статус станции, ПБ до/после, результат и причина, комментарий)
- Out of scope:
  - редактирование/архивирование станции курьером
- Dependencies:
  - Task 1
- Risks:
  - нет
- Definition of done:
  - курьер может войти в систему и видеть только свои разделы
  - курьер видит список только назначенных станций (с кодом, адресом, кол-вом ПБ, слотами, статусом) с сортировкой по количеству ПБ, может открыть карточку (адрес, код, заводской номер, фото, график работы) и выполнить операции с ПБ
  - курьер может отправить комментарий о проблеме по станции; статус операции отображается
  - супер-админ может видеть список курьеров, редактировать данные/статус, блокировать курьера, назначать/удалять станции и просматривать журнал действий

## Порядок поставки

1. Task 1 — backend: учётные записи курьеров, станции для курьера, операции с ПБ.
2. Task 2 — FE: интерфейс курьера и управление курьерами для супер-админа.

## Cross-Repository Notes

- Task 2 (`gozap-admin`) зависит от контракта Task 1 (`gozap-backend`).
- Роль курьера должна быть интегрирована в ролевую модель `sa/docs/admin/admin-roles`.
- Список станций для курьера переиспользует сущность станции из `sa/docs/admin/stations-management`, но с другим набором полей и действий.
