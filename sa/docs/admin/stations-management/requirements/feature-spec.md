## Проблема

- У администратора нет отдельного, целевого раздела для управления станциями (создание новой станции, вывод станции из эксплуатации).
- Доступ к этому разделу должен быть управляем через ролевую модель `sa/docs/admin/admin-roles` (раздел «Станции» в наборе разделов администратора).

## Цель

- Дать администратору (с доступом к разделу «Станции») возможность видеть список станций, создавать новую станцию и архивировать существующую.

## Не входит в текущую итерацию

- Редактирование уже созданной станции (кроме архивирования).
- Удаление станции (только архивирование).
- Привязка станции к партнёру и настройка процента вознаграждения — описано отдельно в `sa/docs/partners/partner-program`.
- Мониторинг технического состояния станции (заряд, количество powerbank в наличии и т.п.) — не упомянуто во входных данных.
- Разархивирование станции — не упомянуто во входных данных.

## Триггер и аудитория

- Точка входа / триггер: администратор с доступом к разделу «Станции» открывает раздел в `gozap-admin`.
- Основная аудитория: администратор с назначенным разделом «Станции» (включая супер-админа).

## Целевое поведение продукта

### Сценарий

1. Администратор с доступом к разделу «Станции» открывает список станций в `gozap-admin`.
2. Администратор создаёт новую станцию.
3. Администратор архивирует станцию, которая больше не используется.

### Правила результата

- Раздел «Станции» должен быть доступен только администратору, которому он назначен (или супер-админу) — см. `sa/docs/admin/admin-roles`.
- Список станций должен отображать все станции, включая архивированные (с явным отличием статуса).
- Создание станции должно добавлять её в список станций, доступных для дальнейшей привязки к партнёру (`sa/docs/partners/partner-program`).
- Архивирование станции не должно приводить к удалению исторических данных о станции (привязки к партнёрам, аренды).

## Бизнес-правила

- Станция — это сущность, создаваемая и управляемая централизованно в разделе «Станции»; привязка станции к партнёру — отдельный процесс (`sa/docs/partners/partner-program`).
- Архивирование — единственное предусмотренное действие по выводу станции из эксплуатации в рамках этой итерации.

## Пограничные сценарии

- Администратор пытается архивировать станцию, которая привязана к партнёру с активными арендами.
- Администратор пытается создать станцию с данными, дублирующими уже существующую станцию.
- Администратор без доступа к разделу «Станции» пытается обратиться к разделу напрямую по ссылке.

## Допущения

- Раздел «Станции» в этой фиче специфицирует только CRUD-операции создания/архивирования и список; детальная карточка станции с техническими параметрами (заряд, прошивка и т.п.) не входит в эту итерацию.
- Сущность «станция», создаваемая в этом разделе, — та же сущность, что привязывается к партнёрам в `sa/docs/partners/partner-program`.
