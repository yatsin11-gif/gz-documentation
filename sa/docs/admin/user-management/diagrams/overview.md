## Управление пользователями в админке — визуальный TL;DR

Источник: [../requirements/feature-spec.md](../requirements/feature-spec.md). Схема — краткая суть, детали в спеке.

### User flow (что видит пользователь)

```mermaid
flowchart TD
    A[Администратор с разделом «Пользователи»] --> LIST[Список пользователей]
    LIST --> CREATE[Создать пользователя]
    CREATE --> CARD[Карточка пользователя]
    LIST --> OPEN[Открыть существующего пользователя]
    OPEN --> CARD
    CARD --> PROMO{Промоаккаунт?}
    PROMO -->|выдать| PROMO_ON[Статус: промоаккаунт]
    PROMO -->|снять| PROMO_OFF[Статус снят]
    CARD --> BLOCK{Заблокировать?}
    BLOCK -->|да| BLOCKED[Статус: заблокирован]
    CARD --> VIEW[Имя, телефон, почта,<br/>текущая аренда, история аренд]
```

### Что внутри (pipeline)

```mermaid
flowchart LR
    REQ[Запрос: создать/промо/блок/карточка] --> CHECK[Backend: раздел «Пользователи» назначен?]
    CHECK -->|нет| DENY[Отклонено]
    CHECK -->|да| OP{Операция}
    OP -->|создание| ACC[(Аккаунт пользователя —<br/>общая сущность со станцией)]
    OP -->|промо| PROMOST[(Статус промоаккаунта)]
    OP -->|блокировка| BLOCKST[(Статус блокировки)]
    BLOCKST -.-> KIOSK[Запрет новой аренды<br/>через станцию]
    OP -->|карточка| CARDDATA[(Имя, телефон, почта,<br/>текущая аренда, история аренд)]
    ACC -.-> SHARED[Общая сущность с<br/>kiosk/terminal-rental-flow]
```
