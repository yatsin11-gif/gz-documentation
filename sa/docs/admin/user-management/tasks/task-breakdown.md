## Список задач

### Task 1

- Title: BE. User Management. Создание пользователя, промоаккаунт, блокировка, карточка
- Repository: `gozap-backend`
- Owner role: Backend engineer
- Зачем нужна задача: администратору нужен API для управления пользователями — той же сущностью, что создаётся автоматически через станцию.
- Scope:
  - создание пользователя администратором (та же сущность аккаунта, что и в `sa/docs/kiosk/terminal-rental-flow`)
  - выдача и снятие статуса промоаккаунта
  - блокировка пользователя (запрет новых аренд через станцию)
  - получение карточки пользователя: имя, телефон, почта, текущая аренда (при наличии), история аренд
  - проверка доступа: только администратор с разделом «Пользователи» или супер-админ
- Out of scope:
  - редактирование данных пользователя после создания
  - удаление пользователя
  - разблокировка пользователя
- Dependencies:
  - `sa/docs/admin/admin-roles` (модель доступа по разделам)
  - `sa/docs/kiosk/terminal-rental-flow` (источник аккаунтов и аренд)
- Risks:
  - эффект промоаккаунта не зафиксирован во входных данных
  - поведение блокировки при активной аренде не зафиксировано
- Definition of done:
  - администратор с доступом к разделу «Пользователи» может создать пользователя, выдать/снять промоаккаунт, заблокировать пользователя и получить карточку пользователя через API

### Task 2

- Title: FE. User Management. Список пользователей, создание, карточка, промоаккаунт и блокировка
- Repository: `gozap-admin`
- Owner role: Frontend engineer
- Зачем нужна задача: администратору нужен интерфейс раздела «Пользователи» — список, форма создания, карточка с действиями.
- Scope:
  - список пользователей
  - форма создания пользователя
  - карточка пользователя (имя, телефон, почта, текущая аренда, история аренд)
  - действия выдачи/снятия промоаккаунта и блокировки из карточки
- Out of scope:
  - редактирование данных пользователя
  - разблокировка пользователя
- Dependencies:
  - Task 1
- Risks:
  - нет
- Definition of done:
  - администратор с доступом к разделу «Пользователи» может увидеть список, создать пользователя, открыть карточку и управлять промоаккаунтом/блокировкой

## Порядок поставки

1. Task 1 — backend API создания пользователя, промоаккаунта, блокировки, карточки.
2. Task 2 — FE раздел «Пользователи».
