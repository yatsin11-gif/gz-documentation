## Test Cases

### TC-001: Код из входящего SMS автоматически подставляется в поле ввода

- Status: Draft
- Type: Functional
- Priority: Low
- Severity: Minor
- Behavior: Positive
- Layer: E2E
- Suite: SMS Autofill
- Related criteria: Автоподстановка кода подтверждения
- Related Jira: `frontend-sms-autofill-code-field.md`
- Qase case: Не назначено
- Tags: `kiosk`, `auth`, `autofill`
- Automation status: manual

#### Preconditions

- Пользователь на экране ввода SMS-кода в рамках основного flow аренды (`terminal-rental-flow`).
- SMS с кодом отправлен на номер пользователя.

#### Test Data

- Номер телефона с возможностью получения SMS на устройстве, поддерживающем автоподстановку.

#### Steps

1. Action: Дождаться доставки SMS с кодом подтверждения после запроса кода.
   Expected result: Код автоматически подставляется в поле ввода без ручного набора.
2. Action: Продолжить flow подтверждения с подставленным кодом.
   Expected result: Подтверждение проходит так же, как при ручном вводе корректного кода.

#### Notes

- Технический механизм автоподстановки на стороне станции не подтверждён (см. coverage gaps).

### TC-002: Ручной ввод кода доступен, если автоподстановка не сработала

- Status: Draft
- Type: Functional
- Priority: Low
- Severity: Minor
- Behavior: Negative
- Layer: E2E
- Suite: SMS Autofill
- Related criteria: Автоподстановка кода подтверждения
- Related Jira: `frontend-sms-autofill-code-field.md`
- Qase case: Не назначено
- Tags: `kiosk`, `auth`, `autofill`
- Automation status: manual

#### Preconditions

- Пользователь на экране ввода SMS-кода.
- Автоподстановка не срабатывает (например, эмулируется отключением механизма автоподстановки).

#### Test Data

- Корректный SMS-код для ручного ввода.

#### Steps

1. Action: Дождаться SMS-кода при отключённой/неуспешной автоподстановке.
   Expected result: Поле ввода кода остаётся доступным для ручного ввода.
2. Action: Ввести код вручную.
   Expected result: Подтверждение проходит успешно, flow продолжается как обычно.

#### Notes

- Резервный сценарий ручного ввода не детализирован в критериях сверх «остаётся доступным» (см. открытые вопросы).

### TC-003: Автоподстановка применяется только к коду текущей сессии на этой станции

- Status: Draft
- Type: Security
- Priority: Medium
- Severity: Major
- Behavior: Negative
- Layer: E2E
- Suite: SMS Autofill
- Related criteria: Правила пользовательского поведения
- Related Jira: `frontend-sms-autofill-code-field.md`
- Qase case: Не назначено
- Tags: `kiosk`, `auth`, `autofill`, `security`
- Automation status: automation candidate

#### Preconditions

- На устройстве пользователя присутствует код от предыдущей/другой сессии аренды (например, на другой станции).
- Текущая сессия на этой станции отправила новый код.

#### Test Data

- Код предыдущей сессии и код текущей сессии на одном устройстве.

#### Steps

1. Action: Дождаться автоподстановки на экране ввода кода текущей сессии при наличии на устройстве кода от другой сессии.
   Expected result: Автоподстановляется только код, отправленный в рамках текущей сессии аренды на этой станции; код другой сессии не используется.
