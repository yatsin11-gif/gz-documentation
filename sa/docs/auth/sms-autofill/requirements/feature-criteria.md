# Критерии

## Источник

- Feature spec: `sa/docs/auth/sms-autofill/requirements/feature-spec.md`
- Утвердил: Не утверждено; draft подготовлен по запросу SA
- Дата: 2026-06-30
- Репозитории в скоупе: `gozap-kiosk-app`

## Цель

- Код подтверждения из входящего SMS должен автоматически подставляться в поле ввода на экране станции в рамках flow подтверждения номера телефона.

## FE Criteria

### Затронутые поверхности

- `gozap-kiosk-app`: экран ввода SMS-кода в рамках `sa/docs/kiosk/terminal-rental-flow`.

### Пользовательские сценарии и состояния

#### Автоподстановка кода подтверждения

- После запроса SMS-кода и получения входящего SMS станция должна автоматически подставить код в поле ввода.
- Если автоподстановка сработала, пользователь не должен набирать код вручную.
- Если автоподстановка не сработала, пользователь должен иметь возможность ввести код вручную (резервный путь не уточнён, см. открытые вопросы).

### Правила пользовательского поведения

- Автоподстановка не должна автоматически подтверждать форму без действия пользователя, если это не уточнено отдельно (открытый вопрос).
- Автоподстановка применяется только к коду, отправленному в рамках текущей сессии аренды на этой станции.

### Ограничения и исключения на FE

- Критерии не фиксируют технический механизм получения SMS на стороне станции.
- Критерии не распространяются на другие поверхности (`gozap-admin`).

## Нефункциональные критерии

### FE

- Производительность: автоподстановка должна происходить без заметной задержки после получения SMS.
- Доступность: поле кода должно оставаться доступным для ручного ввода независимо от попытки автоподстановки.
- Консистентность UX: поведение автоподстановки должно быть одинаковым при каждой повторной отправке кода.

### BE

- Надежность: N/A.
- Наблюдаемость: N/A.
- Безопасность: автоподстановка не должна давать доступ к коду из чужой сессии аренды.

## Вне scope

- Автоподстановка на других поверхностях продукта.
- Создание или изменение Qase test cases.
