## Проблема

- Пользователь на станции должен вручную вводить код из SMS, что замедляет flow аренды у терминала.

## Цель

- Автоматически подставлять код подтверждения из входящего SMS в поле ввода на экране станции.

## Не входит в текущую итерацию

- Автоподстановка кода в кабинете партнёра/админке (`gozap-admin`) — во входных данных не упомянута.
- Поддержка автоподстановки на платформах, отличных от station-терминала.
- Изменение формата или длины SMS-кода.

## Триггер и аудитория

- Точка входа / триггер: пользователь вводит номер телефона в основном flow аренды (`sa/docs/kiosk/terminal-rental-flow`) и ожидает SMS с кодом.
- Основная аудитория: пользователь станции, подтверждающий номер телефона.
- Вторичная аудитория: Не определено.

## Целевое поведение продукта

### Сценарий

1. Пользователь вводит номер телефона на экране станции и запрашивает SMS-код.
2. Станция получает входящее SMS с кодом подтверждения.
3. Код автоматически подставляется в поле ввода без ручного набора пользователем.

### Правила результата

- Код должен подставляться автоматически, если устройство/канал получения SMS это позволяет.
- Если автоподстановка недоступна, пользователь должен иметь возможность ввести код вручную (резервный путь не описан подробно во входных данных).

## Бизнес-правила

- Автоподстановка применяется к коду, отправленному через Twilio в рамках flow подтверждения номера телефона на станции.
