## Приложение станций: основной flow аренды — визуальный TL;DR

Источник: [../requirements/feature-spec.md](../requirements/feature-spec.md). Схема — краткая суть, детали в спеке.

### User flow (что видит пользователь)

```mermaid
flowchart TD
    IDLE[Экран покоя:<br/>номер телефона / альт. кнопка] --> PHONE[Ввод номера]
    IDLE -->|разряженный телефон| ALT[Подсказка:<br/>«Приложите карту для аренды»]
    PHONE --> SMS[Ввод SMS-кода]
    SMS -->|неверный код| ERR1[Ошибка на станции,<br/>аренда не создаётся]
    SMS -->|верный код| ACC[Аккаунт создан]
    ACC --> CARD[Привязка карты + инструкция]
    CARD --> PAY{Оплата успешна?}
    ALT --> PAY
    PAY -->|нет| ERR2[Ошибка на станции,<br/>аренда не создаётся]
    PAY -->|да| OK[Аренда создана<br/>PowerBank выдан]
    OK --> SYNC[Видна в gozap-app<br/>при совпадении номера]
    OK --> NOTIFY[Push если установлено + SMS:<br/>начало аренды]
    NOTIFY -.->|та же логика| ENDN[Push + SMS: завершение аренды<br/>flow завершения не описан]
```

### Что внутри (pipeline)

```mermaid
flowchart LR
    NUM[Номер телефона] --> TWILIO[Twilio: отправка кода]
    TWILIO --> VER[Проверка кода]
    VER -->|успех| ACC[(Создать аккаунт)]
    VER -->|отказ| NOACC[Аренда не создаётся]
    CARD[Данные карты] --> PROV[Платёжный провайдер<br/>не подтверждён]
    PROV -->|отказ| NOREN[Аренда не создаётся]
    PROV -->|успех| RENT[(Создать аренду)]
    RENT --> DISP[Команда станции: выдать PowerBank]
    RENT --> APPSYNC[Синхронизация в gozap-app<br/>по номеру телефона]
    RENT --> NOTIFY[Push не подтверждён + Twilio SMS:<br/>начало аренды]
    ALT[Упрощённый flow: карта без номера] --> PROV
```

> Этап в конвейере: **Jira-ready** (6 issues) → **QA test cases** готовы. См. [../../../PROGRESS.md](../../../PROGRESS.md).

> Вне итерации: окончание аренды и возврат powerbank (flow), экраны `gozap-app`, альтернативные способы оплаты.
