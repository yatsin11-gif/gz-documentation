## Партнёрская программа — визуальный TL;DR

Источник: [../requirements/feature-spec.md](../requirements/feature-spec.md). Схема — краткая суть, детали в спеке.

### User flow (что видит пользователь)

```mermaid
flowchart TD
    A0[Список партнёров:<br/>поиск по логину, сортировка] --> A1[Администратор:<br/>создать партнёра]
    A1 --> A1B[Редактировать данные партнёра]
    A1 --> A2[Привязать станцию + %]
    A2 --> A3{Отвязать станцию?}
    A3 -->|да| A4[Начисления после отвязки<br/>не учитываются]
    A3 -->|нет| A5[Начисления копятся в баланс]
    A4 -->|повторная привязка| A2
    P1[Партнёр: логин] --> P2[Сводка за период<br/>баланс, аренды, выплата]
    P2 --> P3[Детализация по станции]
    P2 --> P4[Кнопка → Stripe:<br/>регистрация/авторизация + карта]
    P4 --> P5[Авто-выплата Stripe<br/>7-го числа каждого месяца]
    P2 --> P7[Кнопка → редирект в Stripe:<br/>история начислений и выплат]
    A6[Администратор:<br/>«как у партнёра»] --> P2
```

### Что внутри (pipeline)

```mermaid
flowchart LR
    RENT[Завершённая аренда на станции] --> CALC[Расчёт доли партнёра<br/>по % на момент завершения]
    CALC --> BAL[(Баланс партнёра)]
    SCHED[Scheduled job 7-го числа:<br/>выплата суммы за предыд. месяц] --> CARD{Счёт/карта указаны?}
    BAL --> CARD
    CARD -->|да| PAY[Stripe payout]
    CARD -->|нет| WAIT[Выплата откладывается,<br/>баланс сохраняется без потери]
    WAIT --> ADDCARD[Партнёр через Stripe-редирект<br/>регистрируется/авторизуется<br/>и указывает счёт/карту] --> PAY
    PAY -->|успех| HIST[(История начислений и выплат<br/>в профиле партнёра в Stripe)]
    PAY -->|ошибка| KEEP[Баланс сохраняется без потери]
    RESET[Запрос сброса пароля] --> TWILIO[Twilio SMS код]
```

> Этап в конвейере: **Jira-ready** (5 issues) → **QA test cases** готовы. См. [../../../PROGRESS.md](../../../PROGRESS.md).

> Вне итерации: ручной запрос выплаты, несколько банковских счетов, несколько валют.
