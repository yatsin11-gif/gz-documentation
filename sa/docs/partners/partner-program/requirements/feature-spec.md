## Проблема

- У бизнеса нет единого инструмента, чтобы заводить владельцев точек (партнёров) и привязывать к ним станции.
- Партнёр не видит свою статистику (станции, аренды, вознаграждение, баланс) и не может получать автоматические выплаты.
- Администратор не может настраивать индивидуальный процент вознаграждения по каждой станции и сверять то, что видит партнёр.

## Цель

- Дать администратору инструмент для создания партнёров, редактирования данных партнёра, привязки/отвязки станций и настройки процента вознаграждения по каждой станции.
- Дать партнёру личный кабинет со статистикой по своим станциям, балансом и автоматическими ежемесячными выплатами через Stripe.

## Не входит в текущую итерацию

- Ручной запрос выплаты партнёром (выплата только автоматическая, раз в месяц).
- Управление договорами/юридическими документами партнёра.
- Множественные банковские счета на одного партнёра.
- Поддержка нескольких валют.
- Уведомления/email-рассылки партнёру (зафиксировано как открытый вопрос ниже).

## Триггер и аудитория

- Точка входа / триггер: администратор создаёт партнёра в админке после офлайн-договорённости; партнёр получает логин/пароль и заходит в свой кабинет.
- Основная аудитория: администратор (создание и управление партнёрами), партнёр (просмотр своей статистики и выплат).
- Вторичная аудитория: супер-админ, которому нужно видеть то же, что видит партнёр, для контроля эффективности точек.

## Целевое поведение продукта

### Сценарий

1. Администратор заводит партнёра (логин, пароль, имя, телефон) и привязывает к нему одну или несколько станций с индивидуальным процентом вознаграждения по каждой станции; при необходимости администратор может отредактировать данные партнёра.
2. Партнёр логинится (с возможностью сброса пароля по SMS) и попадает в раздел, где видны только его данные.
3. Партнёр видит список своих станций и сводку за период: баланс, ближайшую выплату, число завершённых аренд, сумму вознаграждения, разбивку по станциям; может провалиться в конкретную станцию и увидеть список аренд.
4. Система начисляет долю партнёра в баланс по каждой завершённой аренде и 7-го числа каждого месяца автоматически переводит сумму вознаграждения за предыдущий месяц на привязанный счёт партнёра через Stripe.

### Правила результата

- Партнёр должен видеть только данные своих станций, без доступа к чужим.
- Администратор и супер-админ должны видеть ту же статистику, что видит партнёр, для любой выбранной точки.
- Администратор должен иметь возможность отредактировать данные уже созданного партнёра (логин, имя, телефон).
- Баланс партнёра должен учитывать аренду только за период, когда станция была привязана к этому партнёру.
- Сумма вознаграждения за предыдущий месяц должна автоматически выплачиваться 7-го числа каждого месяца без ручного запроса.

## Бизнес-правила

- Процент вознаграждения задаётся отдельно для каждой станции, включая значение 0%.
- При изменении процента вознаграждения на станции аренды, завершённые после момента изменения, считаются по новому проценту; уже начисленные доли по прошлым арендам не пересчитываются.
- Одна станция может быть привязана только к одному партнёру одновременно. После отвязки станции от партнёра аренды, завершённые после момента отвязки, не учитываются в его балансе; после повторной привязки станции аренды, завершённые после момента привязки, снова учитываются в балансе партнёра.
- Доля партнёра рассчитывается системой от суммы каждой завершённой аренды и добавляется в баланс автоматически.
- Поиск в списке партнёров выполняется по логину с частичным совпадением.
- Список партнёров по умолчанию сортируется по дате добавления от новых к старым; администратор/суперадмин может переключить сортировку на логин (по возрастанию/убыванию) или на дату добавления (от новых к старым/от старых к новым).
- Для указания реквизитов выплат партнёр нажимает кнопку и переходит на сторону Stripe: там он регистрируется или авторизуется и вводит данные банковского счёта/карты напрямую в Stripe. Данные карты не вводятся и не хранятся в формах `gozap-admin`/`gozap-backend`.
- Историю начислений и историю выплат партнёр смотрит в своём профиле в Stripe; для просмотра истории начислений партнёр редиректится в Stripe.
- Автоматическая выплата суммы вознаграждения за предыдущий месяц выполняется 7-го числа каждого месяца через Stripe.
- Если на момент плановой выплаты 7-го числа у партнёра не указан банковский счёт/карта, выплата откладывается без потери баланса: как только партнёр указывает счёт/карту, накопленный баланс выплачивается, и далее выплаты продолжаются по расписанию (7-го числа каждого месяца).

## Ограничения и известные лимитации

- Продукт / UX: партнёру не нужно запрашивать выплату вручную — флоу полностью автоматический по расписанию.
- Биллинг / правовые ограничения: для выплат используется Stripe; реквизиты счёта/карты партнёр вводит непосредственно на стороне Stripe через редирект (регистрация/авторизация в Stripe), что указывает на hosted-флоу в духе Stripe Connect; точный Stripe-продукт и порядок верификации партнёра (KYC) выполняются полностью на стороне Stripe и не подтверждены детально на продуктовом уровне.
- Технические ограничения, уже подтвержденные на продуктовом уровне: сброс пароля партнёра выполняется через SMS на номер телефона (используется Twilio).
- Внешние зависимости: Stripe (выплаты партнёрам), Twilio (отправка SMS для сброса пароля).

## Пограничные сценарии

- Администратор отвязывает станцию от партнёра в середине периода — нужно сохранить начисления за период привязки без пересчёта; аренды, завершённые после отвязки, в баланс не попадают.
- Администратор повторно привязывает ранее отвязанную станцию к партнёру — аренды, завершённые после повторной привязки, снова учитываются в балансе.
- Администратор меняет процент вознаграждения на станции при наличии незавершённых аренд — уже начисленные доли сохраняют прежний процент, новые завершения считаются по новому проценту.
- Партнёр не указал банковский счёт/карту к 7-му числу (дате плановой выплаты) — выплата откладывается и выполняется сразу после того, как партнёр пройдёт по кнопке в Stripe и укажет счёт/карту там, далее выплаты продолжаются по расписанию (7-го числа каждого месяца).
- Партнёр пытается открыть историю начислений в `gozap-admin` — система редиректит его в Stripe, где доступна и история начислений, и история выплат.
- У партнёра несколько станций с разным процентом, включая станцию с 0%.
- Поиск в списке партнёров по подстроке логина, не совпадающей ни с одним существующим логином, — список пуст.
- Автоматическая выплата через Stripe завершается ошибкой.
- Партнёр запрашивает сброс пароля, но номер телефона недоступен/не подтверждён.

## Допущения

- Партнёрский кабинет и админка для создания партнёров — это один и тот же веб-продукт `gozap-admin`, в котором роль `partner` видит только свои данные.
- Один партнёр может владеть несколькими станциями, но одна станция привязана только к одному партнёру в любой момент времени.
- Выплата производится 7-го числа каждого месяца суммой вознаграждения за предыдущий месяц.
