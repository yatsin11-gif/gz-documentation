## Список задач

### Task 1

- Title: BE. Partner Program. Учётки партнёров, привязка станций и расчёт баланса
- Repository: `gozap-backend`
- Owner role: Backend engineer
- Зачем нужна задача: без учёток партнёров, привязки станций и расчёта баланса по периоду привязки невозможно построить ни админ-раздел, ни кабинет партнёра.
- Scope:
  - CRUD учётной записи партнёра (логин, пароль, имя, телефон), включая редактирование данных уже созданного партнёра
  - список партнёров с поиском по логину (частичное совпадение) и сортировкой для администратора (по умолчанию по дате добавления от новых к старым; также по логину и по дате добавления по возрастанию/убыванию)
  - привязка/отвязка станций к партнёру с процентом вознаграждения по станции (включая 0%)
  - начисление доли партнёра в баланс при завершении аренды по проценту, действовавшему на станции в момент завершения; процент применяется к базе начисления — реально полученному доходу станции по аренде после обязательных вычетов по платежам (не к полному обороту аренды и не к чистой прибыли); изменение процента применяется только к арендам, завершённым после момента изменения
  - учёт начислений только за период фактической привязки станции к партнёру: аренды, завершённые после отвязки, не учитываются; аренды, завершённые после повторной привязки, снова учитываются
  - эндпоинты для статистики: список станций партнёра, сводка за период, детализация по станции
- Out of scope:
  - интеграция со Stripe для выплат (Task 2)
  - сброс пароля через SMS (Task 3)
- Dependencies:
  - нет (foundation-задача)
- Risks:
  - нет — SA подтвердил: процент, действовавший на станции в момент завершения аренды, фиксируется в начислении и не пересчитывается при последующем изменении процента; отдельного хранения истории процента по периодам не требуется
- Definition of done:
  - администратор может создать партнёра и привязать/отвязать станции с процентом
  - баланс партнёра корректно отражает только аренды за период привязки
  - API отдаёт данные для списка станций, сводки и детализации по станции

### Task 2

- Title: BE. Partner Program. Автоматические ежемесячные выплаты через Stripe
- Repository: `gozap-backend`
- Owner role: Backend engineer
- Зачем нужна задача: сумма вознаграждения за предыдущий месяц должна автоматически выплачиваться 7-го числа каждого месяца без ручного запроса.
- Scope:
  - генерация Stripe-редиректа для регистрации/авторизации партнёра на стороне Stripe и указания им счёта/карты (без хранения данных карты в `gozap-backend`)
  - scheduled job запуска 7-го числа каждого месяца для инициации выплаты суммы вознаграждения за предыдущий месяц через Stripe
  - обработка успешного/неуспешного результата выплаты и обновление баланса
  - отложенная выплата при отсутствии счёта/карты к 7-му числу: накопленный баланс выплачивается сразу после того, как партнёр указывает счёт/карту через Stripe-редирект, далее выплаты продолжаются 7-го числа каждого месяца
  - генерация Stripe-редиректа для просмотра партнёром истории начислений и истории выплат в его профиле Stripe
- Out of scope:
  - выбор конкретного продукта линейки Stripe Connect (Standard/Express/Custom) — требует подтверждения (см. open question)
  - объём KYC/верификации партнёра — выполняется полностью на стороне Stripe
- Dependencies:
  - Task 1 (учётки партнёров и расчёт баланса)
- Risks:
  - точный продукт линейки Stripe Connect не подтверждён
- Definition of done:
  - партнёр может по кнопке перейти в Stripe, зарегистрироваться/авторизоваться и указать счёт/карту
  - выплата суммы вознаграждения за предыдущий месяц выполняется 7-го числа автоматически через Stripe
  - если счёт/карта не указаны к 7-му числу, выплата откладывается и выполняется сразу после указания счёта/карты, далее по расписанию (7-го числа каждого месяца)
  - партнёр может перейти в Stripe для просмотра истории начислений и выплат

### Task 3

- Title: BE. Partner Program. Сброс пароля партнёра через Twilio SMS
- Repository: `gozap-backend`
- Owner role: Backend engineer
- Зачем нужна задача: партнёр должен иметь возможность восстановить доступ к кабинету без участия администратора.
- Scope:
  - генерация и отправка кода сброса пароля через Twilio на номер телефона партнёра
  - проверка кода и установка нового пароля
- Out of scope:
  - сброс пароля для роли администратора (см. `sa/docs/admin/admin-roles`)
- Dependencies:
  - Task 1 (учётка партнёра должна существовать)
- Risks:
  - нет
- Definition of done:
  - партнёр может запросить сброс пароля по SMS и установить новый пароль

### Task 4

- Title: FE. Partner Program. Админ-раздел управления партнёрами
- Repository: `gozap-admin`
- Owner role: Frontend engineer
- Зачем нужна задача: администратору нужен интерфейс для создания партнёров, привязки станций и просмотра представления «как у партнёра».
- Scope:
  - форма создания партнёра
  - форма редактирования данных уже созданного партнёра
  - список партнёров с поиском по логину (частичное совпадение) и сортировкой (по умолчанию по дате добавления от новых к старым; также по логину и по дате добавления по возрастанию/убыванию)
  - интерфейс привязки/отвязки станций с указанием процента
  - переход в представление, идентичное кабинету партнёра, для выбранного партнёра/станции
- Out of scope:
  - сам кабинет партнёра (Task 5, переиспользуется тем же представлением)
- Dependencies:
  - Task 1
- Risks:
  - нет
- Definition of done:
  - администратор может полностью провести партнёра через создание и привязку станций
  - администратор может открыть представление партнёра 1-в-1

### Task 5

- Title: FE. Partner Program. Кабинет партнёра
- Repository: `gozap-admin`
- Owner role: Frontend engineer
- Зачем нужна задача: партнёру нужен личный кабинет со статистикой, балансом и управлением выплатами.
- Scope:
  - логин и сброс пароля партнёра
  - список станций партнёра (код, название, адрес, статус, аренды и вознаграждение за период)
  - сводка за период (баланс, ближайшая выплата, завершённые аренды, вознаграждение, разбивка по станциям)
  - детализация по станции (список аренд: дата, стоимость, вознаграждение + итог)
  - кнопка перехода в Stripe для регистрации/авторизации и указания банковского счёта/карты
  - кнопка перехода (редирект) в Stripe для просмотра истории начислений и истории выплат
- Out of scope:
  - ручной запрос выплаты (не входит в продукт)
  - собственная форма ввода/хранения данных карты и собственная страница истории начислений/выплат — оба пункта реализуются через переход в Stripe
- Dependencies:
  - Task 1, Task 2, Task 3
- Risks:
  - нет
- Definition of done:
  - партнёр видит весь набор статистики и может перейти в Stripe для указания реквизитов выплат и просмотра истории начислений/выплат
  - представление переиспользуется в админ-режиме «как у партнёра» (Task 4)

## Порядок поставки

1. Task 1 — учётки партнёров, привязка станций, расчёт баланса.
2. Task 3 — сброс пароля через Twilio (можно параллельно с Task 2).
3. Task 2 — интеграция со Stripe для выплат.
4. Task 4 и Task 5 — FE админ-раздел и кабинет партнёра (после готовности Task 1, далее достраиваются по мере готовности Task 2/3).

## Cross-Repository Notes

- Task 4 и Task 5 переиспользуют один и тот же UI-слой представления статистики партнёра в `gozap-admin`, отличаясь только правами доступа (admin vs partner).
- Расчёт баланса (Task 1) является зависимостью для ролевой модели из `sa/docs/admin/admin-roles`, если уровень доступа администратора будет ограничивать видимость партнёрского раздела.
